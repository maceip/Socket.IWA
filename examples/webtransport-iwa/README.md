# WebTransport IWA Example

A complete, self-contained WebTransport echo server and client built on the same
ngtcp2 + nghttp3 + wolfSSL stack as the root `quic_echo_server.c`.

## What it does

- **`server.c`** — QUIC + HTTP/3 + WebTransport server that:
  - Accepts WebTransport sessions via Extended CONNECT (`:protocol=webtransport`)
  - Echoes bidirectional stream data back to the client
  - Echoes DATAGRAM frames back to the client
  - Responds to HTTP/3 GET on `/`, `/wt`, `/.well-known/webtransport`

- **`client.html`** — Browser client using `new WebTransport()` that:
  - Connects with optional `serverCertificateHashes` for self-signed certs
  - Sends/receives datagrams
  - Opens bidirectional streams and reads echo responses
  - Accepts incoming server-initiated streams

## Quick start (native build)

```bash
# 1. Generate a self-signed certificate (outputs cert_data.h)
make cert_data.h
# Note the base64 SHA-256 hash printed — you'll paste it in the client

# 2. Build the native server (requires ngtcp2, nghttp3, wolfSSL installed)
make native

# 3. Start the server
./wt_echo_server
# Listens on UDP 0.0.0.0:4433

# 4. Open client.html in Chrome (115+)
#    - Paste the base64 cert hash into the "Cert hash" field
#    - Click "Connect"
#    - Send datagrams or stream messages
```

## WASM build (for IWA deployment)

The WASM build uses Emscripten with the Direct Sockets patch from this repo.
Run inside the Docker build environment (see `../../docker_build_quic.sh`):

```bash
make wasm INSTALL_PREFIX=/build/deps
```

This produces `wt_echo_server.js` + `wt_echo_server.wasm` which can be loaded
from an IWA's `index.html` (see `../../iwa/public/index.html` for the pattern).

## Architecture

```
Chrome (WebTransport API)          Server (this example)
─────────────────────────          ─────────────────────
new WebTransport(url, {            UDP socket on :4433
  serverCertificateHashes: [...]       │
})                                     ▼
    │                              ngtcp2 (QUIC)
    ▼                                  │
QUIC handshake (TLS 1.3)              ▼
    │                              wolfSSL (TLS 1.3)
    ▼                                  │
HTTP/3 Extended CONNECT                ▼
  :method = CONNECT                nghttp3 (HTTP/3)
  :protocol = webtransport             │
  :path = /wt                          ▼
    │                              h3_end_headers()
    ▼                              detects CONNECT + webtransport
Server responds 200                    │
    │                                  ▼
    ▼                              Echo loop:
Datagrams ◄──────────────────────► recv_datagram_cb → echo back
Bidi streams ◄───────────────────► h3_recv_data → buffer → write_streams
```

## Notes

- The server accepts one connection at a time (for simplicity).
- Certificate validity is 14 days. Re-run `make cert_data.h` to regenerate.
- For IWA contexts, Direct Sockets permissions grant raw UDP access,
  so `serverCertificateHashes` is used instead of a CA-signed certificate.
- The `sec-webtransport-http3-draft: draft02` header is sent for compatibility
  with Chrome's WebTransport implementation.
